Program Overview

From occasional Claude users
to systematic operators.

Welspun One already has Claude deployed company-wide. This masterclass does not start from zero. It picks up where access ends and capability begins. Every module is hands-on, demonstration-first, and built around real business scenarios from logistics, supply chain, analytics, and corporate functions.

Most employees with Claude access are using 20% of what it can do. This masterclass closes that gap, not with theory, but with workflows participants can run the next morning.

Who This Is For

Employees across Operations, Strategy, Business Development, Supply Chain, Analytics, and Corporate Functions. Participants already have paid Claude accounts and working familiarity with the tool.

What This Is Not

This is not an AI awareness session. No time spent on what AI is or why it matters. Every minute is practical, demonstrated, and immediately applicable to day-to-day work.

Day 1 Focus

Prompt Engineering, Memory, Projects, and Skills. Participants leave Day 1 with a functioning Claude Project and one live Skill built during the session.

Day 2 Focus

Curriculum

7 Modules. 2 Days.

Day 1 builds the foundation. Day 2 applies it to real tools, real data, and automated workflows. Every module ends with a hands-on exercise participants complete before moving forward.

01

How Claude Works, Prompt Engineering, and Getting Finished Work Out of It

Reset how participants think about Claude and give them a repeatable prompting framework they can apply across every function immediately.

Topics Covered
  • Why most paid Claude users are stuck at 20% of its capability. The delegation mindset vs. the prompt mindset
  • Claude's three products: Chat, Cowork, and Code. Which one to use and when
  • Context window and tokens. Why the way you load information determines output quality
  • Data privacy on paid accounts. Why Welspun's data never trains Claude's models
  • The four-part brief structure. Role, context, task, output format
  • Constraint prompting. What to include, what to explicitly exclude
  • Iterative refinement in three turns. How to fix bad outputs without starting over
  • Live exercise: Participants rewrite three of their own real prompts using the framework
Outcome

Participants understand Claude's architecture, choose the right product for the right task, and have a personal prompt framework built from their own work tasks.

Tools Covered
Claude.ai Claude Cowork Claude Code
02

Memory and Projects. Making Claude Remember Your Business

Eliminate the starting-from-zero problem by building persistent memory structures for teams and functions.

Topics Covered
  • Global instructions. What to set once so every Claude conversation starts right
  • Claude Projects. How to create one per team, function, or client
  • Writing project instructions that make outputs consistent without re-explaining every time
  • Uploading context files. SOPs, style guides, org structure, data references
  • Live exercise: Each participant sets up one Project for their function with instructions and one uploaded file
Outcome

Every participant has a functioning Claude Project with written instructions and context loaded before Day 1 ends.

Tools Covered
Claude.ai Claude Projects
03

Skills. Claude's Muscle Memory

Participants understand what Skills are, when to build one, and build their first Skill using the TAR framework during the session.

Topics Covered
  • What a Skill is and how it differs from a Project or a prompt
  • The TAR framework. Trigger, Action, Result. Planning before building
  • When to build a Skill vs. when to just prompt. The 6-point decision checklist
  • Anatomy of a Skill folder. SKILL.md, reference files, scripts, assets
  • Writing skill descriptions that fire precisely. Good vs. bad description examples
  • Live build: Participants use Skill Creator to build one Skill for a task they repeat at least three times a week
Outcome

Every participant leaves Day 1 with one working Skill built around a real recurring task from their own job.

Tools Covered
Claude.ai Claude Skills Skill Creator
04

Claude for Excel. Deep Dive

Transform how participants use Excel by making Claude an active analysis partner for data work, not just a formula lookup tool.

Topics Covered
  • Formula generation and live debugging. Describe the logic, get the formula
  • Data cleaning in plain English. Inconsistent entries, duplicates, column restructuring
  • Analysis from raw data. Trends, comparisons, anomalies, period-over-period breakdowns
  • Building structured Excel reports. Commentary, flagged metrics, executive summaries
  • Handling large files. Python-assisted chunking for datasets beyond standard paste limits
  • Business use case examples: Logistics cost analysis, warehouse inventory variance, vendor performance tracking
  • Live exercise: Participants bring one real Excel file and run three analysis tasks using Claude
Outcome

Participants can delegate formula work, data cleaning, and report commentary to Claude and get accurate, structured outputs back.

Tools Covered
Claude.ai Microsoft Excel Claude for Excel
05

Business Document and Report Production

Replace slow manual drafting of standard corporate documents with Claude-assisted workflows that produce first drafts in minutes.

Topics Covered
  • SOPs and process documentation. Speak the process, Claude structures the document
  • Business reports and executive summaries. Raw data or notes in, formatted report out
  • Presentation structuring. From a bullet-point brain dump to a slide narrative
  • Research synthesis. Upload multiple sources, get one consolidated briefing document
  • Email and escalation drafting. Tone-matched, context-aware, ready to send
Outcome

Participants produce first drafts of SOPs, reports, presentations, and emails in a fraction of the current time using a repeatable workflow.

Tools Covered
Claude.ai Google Drive Microsoft Word PowerPoint
06

MCP Connectors. Connecting Claude to Your Real Tools

Participants understand what MCP is, connect at least one live tool during the session, and see Claude reading their actual data.

Topics Covered
  • What MCP is and why it matters. From copy-paste to live data access in real time
  • How to connect a tool. Step-by-step setup inside Claude Desktop
  • Permissions and safety. What you allow Claude to read vs. write
  • Tools most relevant to Welspun functions: Google Drive, Google Sheets, Gmail, Google Calendar
  • Live setup: Each participant connects one MCP tool and runs a test prompt against their live data
  • What becomes possible after MCP. Removing the manual middle step from recurring workflows
Outcome

Every participant has at least one MCP tool connected and has seen Claude pull live data from it during the session.

Tools Covered
Claude Desktop MCP Connectors Google Drive Google Sheets Gmail
07

Workflow Automation and Scheduling

Show participants how to make Claude workflows run automatically without them initiating anything, turning one-time setups into ongoing systems.

Topics Covered
  • How scheduling works in Claude Desktop. Setup, frequency, output folders
  • Designing a scheduled workflow. What tasks are worth automating vs. what need human judgment
  • Examples relevant to corporate functions: Daily data summaries, weekly report generation, recurring email drafts
  • The Pipeline Method folder structure. Raw, Context, Deliverables, Archive
  • Live demo: Build and schedule one automated task end-to-end during the session
Outcome

Participants can design and set up a scheduled Claude workflow for at least one recurring task in their function.
